1 Зачем нужен GIT и какие задачи решают с его помощью?

Git нужен чтобы иметь возможность контролировать, управлять ,переключатся между версиями проэкта или документов, возвращатся в любой момент любой версии если содержимое не соответствует ожиданиям и с этой точки писать заново, Так же git позволяет работать над одним проэктом одновременно многим учасникам,

2 Что такое репозиторий GIT и где он находится?

Это папка (директория) внутри которой все файлы и папки, история их изменения, и друг служ инф, находится под контролем и с которими могут проводится необходимые пользователю манипуляции. Может находится в любом месте на ПК , так же может быть удаленным

3 Для чего используется файл .gitignore?

В нем указываются файлы и папки которые git не будет контролировать (отслеживать ). То есть будет игнорировать. Обращатся к ним можна по разному , как указав сам файл , так и указав тип файла. В таком случае git будет игнорировать все файлы этого типа

4 Что такое "коммит"?

Это точка сохранения в которой инф о всех изменениях начиная от предыдущего коммита. и др служ инф. Возможно не совсем правильно но у меня асоциация с сохранением в компьютерной игре , Чтото пошло не так - запускаешь с последнего или любого другого сохранения.

Укажите базовые команды в рамках работы с GIT:

Настройка аккаунта локального пользователя.

git config --global user.name ”Anatolii Chumak”

git config --global user.email "Tolik4umak@gmail.com"

Создание репозитория.

git init

Добавление файлов в индекс и выполнение коммита.

git add имя файла
git add . - добавит все файлы в папке

Просмотр журнала коммитов.

git log
